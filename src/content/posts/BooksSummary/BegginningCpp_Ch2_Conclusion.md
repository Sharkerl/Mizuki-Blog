# 第2章：基础数据类型入门

<a id="变量数据与数据类型"></a>

## 变量、数据与数据类型

**变量(Variable)** 是你定义的一块带有名称的内存。每个变量只存储特定类型的数据。每个变量都有一个**类型(Type)**，决定了它可以存储什么样的数据。每种 **基本类型(Fundamental Type)** 都由一个唯一的 **类型名(Type Name)** 标识，该类型名由一个或多个 **关键字(Keyword)** 组成——关键字是 C++ 中的保留字，你不能将其用作其他用途。

编译器会进行大量检查，确保你在任何上下文中都使用了正确的数据类型。它还确保当你将不同类型组合在一个操作中（比如给两个值相加）时，它们要么是同一类型，要么能够通过将一个值转换为另一个值的类型来实现兼容。编译器会检测并报告尝试组合不兼容数据类型的行为。

数值可以分为两大类：**整数(Integer)**（完整的数字）和**浮点数(Floating-Point Value)**（可以是非整数的数字）。每一类都有多种 C++ 基本类型，每种类型可以存储一个特定范围的数值。我们从整数类型开始。

### 定义整数变量

下面是一条定义整数变量的语句：

```cpp
int apple_count;
```

这定义了一个名为 `apple_count` 的 `int` 类型变量。该变量将包含某个任意的垃圾值。你应该在定义变量时就指定一个初始值：

```cpp
int apple_count {15}; // 苹果的数量
```

`apple_count` 的初始值出现在花括号内，这种语法称为**花括号初始化(Braced Initializer)**。稍后你会遇到花括号内有多个值的情况。虽然你不一定要在定义变量时进行初始化，但这是一个好习惯。确保变量以已知值开始，这样当代码不按预期工作时更容易排查问题。

`int` 类型变量的大小通常为四个字节，可以存储从 -2,147,483,648 到 +2,147,483,647 的整数。这涵盖了大多数场景，这也是 `int` 成为最常用的整数类型的原因。

你可以在一条语句中定义和初始化多个变量：

```cpp
int apple_count {15}; // 苹果的数量
int orange_count {5}; // 橙子的数量
int total_fruit {apple_count + orange_count}; // 水果总数
```

`total_fruit` 的初始值展示了初始值可以是一个表达式——表达式中引用的变量必须在源代码文件中更早地定义。

还有另外两种初始化变量的方式：**函数式表示法(Functional Notation)**和**赋值表示法(Assignment Notation)**：

```cpp
int lemon_count(4); // 函数式表示法
int tomato_count = 12; // 赋值表示法
```

大多数时候，这三种表示法——花括号、函数式和赋值表示法——是等价的。然而，在所谓的**窄化转换(Narrowing Conversion)**方面，花括号初始化形式稍微更安全一些。窄化转换将一个值转换为范围更有限的类型，有可能丢失信息。例如：

```cpp
int banana_count(7.5); // 通常编译无警告，banana_count = 7
int tangerine_count = 5.3; // 通常编译无警告，tangerine_count = 5
int papaya_count{0.3}; // 编译器警告或错误：窄化转换
```

正常情况下，你提供的初始值应该与你正在定义的变量类型相同。但如果不同，编译器会尝试将其转换为所需类型。在前面的例子中，我们为两个整数变量指定了非整数初始值。在这两个变量定义之后，`banana_count` 和 `tangerine_count` 将分别包含整数值 7 和 5。这可能并不是作者的本意。

尽管如此，就 C++ 标准而言，这两条定义是完全合法的。它们允许编译而不发出任何警告。虽然有些编译器会对这种明目张胆的窄化转换发出警告，但并非所有编译器都会这样做。如果你使用花括号初始化形式，符合标准的编译器则至少会发出诊断消息。例如：

```cpp
int papaya_count{0.3}; // 至少会有编译器警告，通常会报错
```

如果这条语句编译成功，`papaya_count` 将被初始化为整数值 0。但至少编译器已经给了你一个警告，告诉你可能出了问题。有些编译器甚至会直接报错并拒绝编译这样的定义。

我们认为，无意间的窄化转换不应该被忽视，因为它们往往是一种错误。在本书中，我们将采用花括号初始化语法。这是 C++11 专门引入以标准化初始化的最新语法。除了在为窄化转换提供更好的安全保障之外，它的主要优势在于让你能够以相同的方式初始化几乎所有东西——因此它也被称为**统一初始化(Uniform Initialization)**。

> :::tip
> 你可以在一条语句中定义和初始化多个给定类型的变量。例如：
>
> ```cpp
> int foot_count {2}, toe_count {10}, head_count {1};
> ```
>
> 虽然这是合法的，但通常建议每个变量单独定义。这样代码更易读，你也可以为每个变量添加注释说明其用途。
> :::

你可以在标准输出流中写入任何基本类型变量的值。下面是一个用几个整数演示的程序：

```cpp
// Ex2_01.cpp
// 将变量值输出到屏幕
import <iostream>; // 用于通过 std::cin / cout 进行用户输入和输出

int main()
{
    int apple_count {15}; // 苹果的数量
    int orange_count {5}; // 橙子的数量
    int total_fruit {apple_count + orange_count}; // 水果总数

    std::cout << "apple_count 的值为 " << apple_count << std::endl;
    std::cout << "orange_count 的值为 " << orange_count << std::endl;
    std::cout << "total_fruit 的值为 " << total_fruit << std::endl;
}
```

如果你编译并执行这个程序，你会看到它输出了三个变量的值，并附带了一些解释它们是什么的文本。整数值通过插入运算符 `<<` 自动转换为字符表示。这适用于任何基本类型的值。

> :::tip
> Ex2_01.cpp 中的三个变量当然不需要任何注释来解释它们代表什么。它们的变量名已经说得很清楚了——变量名本就该如此！相比之下，一个不太好的程序员可能会写出以下内容：
>
> ```cpp
> int n {15};
> int m {5};
> int t {n + m};
> ```
>
> 没有额外的上下文或解释，没有人能猜出这段代码是在统计水果。因此，你应该始终选择尽可能**自我描述(Self-descriptive)**的变量名。正确命名的变量和函数大多不需要额外的注释解释。当然，这并不意味着你不应该在声明中添加注释。你不可能用一个名字捕捉所有内容。几句或必要时一段话的注释可以大大帮助他人理解你的代码。写作时多花一点精力，可以显著加快未来的开发速度！
> :::

<a id="有符号整数类型"></a>

## 有符号整数类型

下表展示了存储**有符号整数(Signed Integer)**（即正值和负值）的基本类型的完整集合：

**表 2-1. 有符号整数类型**

| 类型名 | 典型大小（字节） | 典型值范围 | 理论最小大小 |
| --- | --- | --- | --- |
| `signed char` | 1 | -128 到 +127 | 1 |
| `short` / `short int` / `signed short` / `signed short int` | 2 | -32,768 到 +32,767 | 2 |
| `int` / `signed` / `signed int` | 4 | -2,147,483,648 到 +2,147,483,647 | 2 |
| `long` / `long int` / `signed long` / `signed long int` | 4 或 8 | 同 `int` 或 `long long` | 4 |
| `long long` / `long long int` / `signed long long` / `signed long long int` | 8 | -9,223,372,036,854,775,808 到 +9,223,372,036,854,775,807 | 8 |

`signed char` 类型始终占一个字节（而一个字节几乎总是八位）；其他类型占用的字节数取决于编译器。不过，每种类型占用的内存至少与列表中 preceding 的类型一样多，并且（截至 C++20）至少如表 2-1 最后一列所示。

如果第一列中出现两个类型名，优先使用前面那个缩写名。也就是说，你通常会看到 `long` 而不是 `long int` 或 `signed long int`。

`signed` 修饰符大多数情况下是可选的——如果省略，你的类型默认就是有符号的。唯一的例外是 `char`。虽然未修改的 `char` 类型确实存在，但它是编译器相关的——取决于编译器，它可能是有符号的也可能是无符号的。我们将在后面讨论这个问题。对于除 `char` 之外的所有整数类型，你可以自由选择是否添加 `signed` 修饰符。个人而言，我们通常只在真正想强调某个特定变量是有符号的时候才会添加它。

> :::tip
> `signed` 被认为是 `signed int` 的简写。缩写名更常用（例如 `long` 而非 `long int`）。
> :::

<a id="无符号类型"></a>

## 无符号类型

当然，有些情况下你不需要存储负数。一个班级里的学生人数或一个装配线上的零件数量总是正整数。你可以通过在任何有符号整数类型名前加上 `unsigned` 关键字来指定只存储非负值的整数类型——例如 `unsigned char`、`unsigned short` 或 `unsigned long long`。每个无符号类型都是有符号类型的一种不同类型，但占用相同的内存量。

与其他整数类型不同，`char` 类型是一种不同于 `signed char` 和 `unsigned char` 的独立整数类型。`char` 类型仅用于存储字符代码的变量，根据你的编译器，它可能是有符号的或无符号的类型。

> :::tip
> 只使用未修改的 `char` 类型变量来存储字母字符。要存储数字，你应该使用 `signed char` 或 `unsigned char`。如果你发现自己需要存储、引用或操作原始二进制数据（本书不讨论的高级主题），你应该使用 C++20 的 `std::byte` 类型，而不是传统的 `char` 或 `unsigned char`。
> :::

除 `unsigned char`（可能的值为 0 到 255）以及可能的 `unsigned short`（通常为 0 到 65,535）外，增加可表示数字的范围很少是添加 `unsigned` 修饰符的主要动机。事实上，无论你能表示到 2,147,483,647 还是 4,294,967,295，差别通常不大。相反，你主要添加 `unsigned` 修饰符来使代码更具**自文档性(Self-documenting)**，让代码更可预测，明确某个变量将或应该包含什么值。

> :::note
> 你也可以单独使用 `signed` 和 `unsigned` 关键字。如表 2-1 所示，`signed` 被认为是 `signed int` 的简写。因此，`unsigned` 自然是 `unsigned int` 的简写。
> :::

<a id="零初始化"></a>

## 零初始化

下面这条语句定义了一个初始值等于零的整数变量：

```cpp
int counter {0};  // counter 从零开始
```

你可以省略花括号初始化器中的 0，效果是一样的。因此 `counter` 的定义可以写成这样：

```cpp
int counter {};   // counter 从零开始
```

空的花括号在某种程度上类似于数字零，使得这种语法很容易记住。**零初始化(Zero Initialization)**适用于任何基本类型。例如，对于所有基本数值类型，空的花括号初始化器总是被认为包含数字零。

<a id="定义固定值变量"></a>

## 定义固定值变量

有时你想定义值固定且不得更改的变量。你在变量定义中使用 `const` 关键字。这类变量通常称为**常量(Constant)**。

下面是一个示例：

```cpp
const unsigned toe_count {10}; // 固定值为 10 的无符号整数
```

`const` 关键字告诉编译器 `toe_count` 的值不得更改。任何试图修改此值的语句都将在编译期间被标记为错误——切断一个人的脚趾绝对不行！你可以使用 `const` 关键字来固定任何类型变量的值。

> :::tip
> 如果不考虑其他因素，知道哪些变量的值可以更改、哪些不能更改，会让你的代码更易于理解。因此，我们建议在适用时添加 `const` 修饰符。
> :::

<a id="整数字面量"></a>

## 整数字面量

任何形式的常量值，如 `42`、`2.71828`、`'Z'` 或 `"MarkTwain"`，都称为**字面量(Literal)**。这些例子依次是：**整数字面量(Integer Literal)**、**浮点字面量(Floating-Point Literal)**、**字符字面量(Character Literal)**和**字符串字面量(String Literal)**。每个字面量都属于某种类型。我们先解释整数字面量，其他类型的字面量将在后续内容中介绍。

### 十进制整数字面量

你可以非常直接地编写十进制整数字面量。以下是一些十进制整数的示例：

```
-123L  +123  123  22333  98U  -1234LL  12345ULL
```

无符号整数字面量附加 `u` 或 `U`。`long` 类型和 `long long` 类型的字面量分别附加 `L` 或 `LL`，如果它们是无符号的，还附加 `u` 或 `U`。如果没有后缀，整数常量的类型为 `int`。`U` 和 `L` 或 `LL` 的顺序可以互换。你可以使用小写的 `L` 和 `LL` 后缀，但我们建议你不要这样做，因为小写的 `l` 很容易与数字 `1` 混淆。

你可以省略第二个示例中的 `+`，因为它默认隐含。但如果你觉得加上它能让事情更清晰，那也没问题。字面量 `+123` 与 `123` 相同，类型为 `int`，因为没有后缀。

第四个示例 `22333` 这个数字，根据当地惯例，你可能写成 `22,333`、`22 333` 或 `22.333`（尽管也存在其他格式约定）。不过在 C++ 整数字面量中，你不得使用逗号或空格，添加一个点会使其变成浮点字面量（稍后讨论）。然而，自 C++14 起，你可以使用单引号字符 `'` 来提高数值字面量的可读性。例如：

```cpp
22'333  -1'234LL  12'345ULL
```

下面是一些使用这些字面量的语句：

```cpp
unsigned long age {99'000'000UL}; // 99ul 或 99LU 也可以
unsigned short price {10u}; // 没有特定的 short 字面量类型
long long distance {15'000'000LL}; // 1500万的常见分组
```

请注意，对数字分组没有任何限制。大多数西方惯例每三位一组，但这并非普遍适用。例如，印度次大陆的本地人通常会这样写 1500 万（除最右边三位一组外，其余每两位一组）：

```cpp
1'50'00'000LL
```

到目前为止，我们在变量初始化中非常努力地添加字面量后缀——`u` 或 `U` 表示无符号，`L` 表示 `long` 类型等等。但实际上，在这种形式的变量初始化中，你很少会添加这些后缀。原因是编译器永远不会对你直接这样写提出抱怨：

```cpp
unsigned long age {99};
unsigned short price {10}; // 没有特定的 short 字面量类型
long long distance {15'000'000}; // 1500万的常见分组
```

虽然所有这些字面量技术上都是（有符号）`int` 类型，但你的编译器会愉快地将它们转换为你需要的正确类型。只要目标类型可以表示给定值而不丢失信息，就没有必要发出警告。

> :::note
> 虽然大多数时候是可选的，但有些情况下你确实需要添加正确的字面量后缀，例如当你用 `auto` 类型初始化变量时（本章末尾会解释），或在调用以字面量为参数的重载函数时（第 8 章覆盖）。
> :::

初始化值应始终在变量类型的允许范围内，并且来自正确的类型。下面两条语句违反了这些限制。也就是说，它们需要窄化转换：

```cpp
unsigned char high_score { 513U }; // unsigned char 的有效范围为 [0, 255]
unsigned int high_score { -1 }; // -1 是 signed int 类型的字面量
```

正如我们之前解释的，这两条花括号初始化语句至少会产生编译器警告，如果用的是某些编译器，甚至会导致编译错误。

### 十六进制字面量

你可以将整数字面量写成十六进制值。十六进制字面量以 `0x` 或 `0X` 为前缀，因此 `0x999` 是一个类型为 `int`、包含三个十六进制数字的十六进制数。而普通的 `999` 是一个类型为 `int` 的十进制值，所以它们的值完全不同。以下是一些十六进制字面量的更多示例：

| 十六进制字面量 | 十进制字面量 |
| --- | --- |
| `0x1AF` | `431` |
| `0x123U` | `291U` |
| `0xFF` | `255` |

十六进制字面量的一个重要用途是定义特定的**位模式(Bit Pattern)**。每个十六进制数字对应四个位，因此很容易用十六进制字面量来表达位模式。例如，像素颜色的红、绿、蓝分量（RGB 值）通常表示为打包到 32 位字中的三个字节。白色可以用 `0xFFFFFF` 指定，因为白色的三个分量强度都具有相同的最大值 255，即 `0xFF`。红色则为 `0xff0000`。

```cpp
unsigned int color {0x0ff1ce}; // 无符号整数十六进制常量 - 十进制 1,044,942
int mask {0xFF00FF00}; // 四个字节的指定：FF, 00, FF, 00
```

### 八进制字面量

你还可以将整数字面量写成八进制值——即以 8 为基数。你通过以零开头来标识一个八进制数。

| 八进制字面量 | 十进制字面量 |
| --- | --- |
| `0657` | `431` |
| `0443U` | `291U` |
| `0377` | `255` |

> :::caution
> 不要以零开头的十进制整数值。编译器会将此类值解释为**八进制(Base 8)**。例如，`065` 会被解释为八进制（十进制 53），而不是十进制 65。
> :::

### 二进制字面量

二进制字面量由 C++14 标准引入。你将二进制整数字面量写成由二进制数字（0 或 1）组成的序列，并以 `0b` 或 `0B` 为前缀。与往常一样，二进制字面量可以附加 `L` 或 `LL` 来表示它是 `long` 或 `long long` 类型，如果是无符号字面量则附加 `u` 或 `U`。

| 二进制字面量 | 十进制字面量 |
| --- | --- |
| `0b110101111` | `431` |
| `0b100100011U` | `291U` |
| `0b11111111` | `255` |

对于编译器来说，无论你选择哪种基数来编写整数值，最终它都会被存储为二进制数。编写整数的不同方式只是为了方便。你选择一个或另一个可能的表示方式来适应上下文。

> :::note
> 你可以在任何整数字面量中使用单引号作为分隔符以提高可读性。这包括十六进制或二进制字面量。例如：
>
> ```cpp
> 0xFF00'00FFu
> 0b11001010'11011001
> ```
> :::

<a id="整数运算"></a>

## 整数运算

首先，我们来理清一些术语。像加法或乘法这样的运算由**运算符(Operator)**定义——加法和乘法的运算符分别是 `+` 和 `*`。运算符作用的值称为**操作数(Operand)**，因此在表达式 `2*3` 中，操作数是 2 和 3。需要两个操作数的运算符称为**二元运算符(Binary Operator)**。需要一个操作数的运算符称为**一元运算符(Unary Operator)**。一元运算符的一个例子是表达式 `-width` 中的减号。减号取 `width` 值的相反数，因此表达式的结果是与操作数符号相反的值。这与二元乘法运算符形成对比，后者作用于两个操作数。

**表 2-2. 基本算术运算**

| 运算符 | 运算 |
| --- | --- |
| `+` | 加法(Addition) |
| `-` | 减法(Subtraction) |
| `*` | 乘法(Multiplication) |
| `/` | 除法(Division) |
| `%` | 取模(Modulus，除法后的余数) |

表 2-2 中的所有运算符都是二元运算符，工作方式大致如你所料。有两个运算符可能需要稍作解释：可能较少为人知的取模运算符，以及除法运算符。C++ 中的**整数除法(Integer Division)** 有点特殊。当应用于两个整数操作数时，除法运算的结果始终是一个整数。

假设你写：

```cpp
int numerator = 11;
int quotient = numerator / 4; // quotient = 2（不是 2.75）
```

从数学上讲，11/4 的结果当然是 2.75。但 2.75 显然不是整数，那该怎么办？任何理智的数学家都会建议你将商四舍五入到最近的整数，即 3。但是，你的计算机不会这样做。相反，你的计算机会简单地完全丢弃小数部分 0.75。大概是因为正确的四舍五入需要更复杂的电路，因此也需要更多时间来评估。这意味着在 C++ 中，11/4 始终给出整数值 2。

**取模运算符(Modulus Operator)** `%` 补充了除法运算符，它产生整数除法后的**余数(Remainder)**。它定义为：对于所有整数 `x` 和 `y`，`(x / y) * y + (x % y)` 等于 `x`。使用这个公式，你可以轻松推导出取模运算符对负操作数做什么。

> :::caution
> 除法和取模运算符在右操作数为零时的结果是**未定义的(Undefined)**——也就是说，会发生什么取决于你的编译器和计算机架构。始终检查你的除数！
> :::

<a id="复合算术表达式"></a>

## 复合算术表达式

当多个运算符出现在同一表达式中时，乘法、除法和取模运算总是在加法和减法之前执行。例如：

```cpp
long width {4};
long length {5};
long perimeter {(width + length) * 2}; // 结果为 18
long perimeter {width + length * 2};    // 结果为 14（不同！）
```

你可以使用**括号(Parentheses)**来控制更复杂表达式的执行顺序。括号内的子表达式首先求值，然后将结果乘以二，产生与之前相同的结果 18。但是如果省略括号，结果就不再是 18，而是 14：

```cpp
long perimeter {width + length * 2}; // 结果为 14
```

原因是乘法总是在加法之前求值。因此，前面的语句实际上等价于：

```cpp
long perimeter {width + (length * 2)};
```

括号可以嵌套，在这种情况下，括号之间的子表达式从内到外按顺序执行。这个带有嵌套括号的表达式示例展示了它是如何工作的：

```cpp
2 * (a + 3 * (b + 4 * (c + 5 * d)))
```

表达式 `5*d` 首先求值，然后将 `c` 加到结果上。该结果乘以 4，然后加上 `b`。该结果乘以 3，然后加上 `a`。最后，该结果乘以 2，产生整个表达式的结果。

> :::tip
> **始终添加括号以提高清晰度**，即使默认的求值顺序碰巧正是你想要的。例如：
>
> ```cpp
> long perimeter {(2 * width) + (2 * length)}; // 结果为 18
> ```
> :::

<a id="赋值运算"></a>

## 赋值运算

在 C++ 中，只有在使用 `const` 限定符时变量的值才是固定的。在其他所有情况下，变量的值总是可以被新值覆盖：

```cpp
long perimeter {};
// ...
perimeter = 2 * (width + length);
```

最后一行是一个**赋值语句(Assignment Statement)**，`=` 是**赋值运算符(Assignment Operator)**。赋值运算符右侧的算术表达式被求值，结果存储在左侧的变量中。虽然在声明时初始化 `perimeter` 变量可能不是严格必要的——只要变量在被赋值之前不被读取就行——但始终初始化你的变量被认为是良好的做法。而零通常和任何其他值一样好。

你可以在一条语句中将值赋给多个变量：

```cpp
int a {}, b {}, c {5}, d {4};
a = b = c * c - d * d; // b = 9，然后 a = 9
```

第二条语句计算表达式 `c*c - d*d` 的值并将其存储在 `b` 中，因此 `b` 将被设为 9。接下来 `b` 的值被存储在 `a` 中，因此 `a` 也将被设为 9。你可以有任意多次重复赋值。

重要的是要认识到，赋值运算符与代数方程中的等号截然不同。后者意味着相等，而前者指定的是一个**动作(Action)**——具体来说，是覆盖给定内存位置的行为。变量可以被覆盖任意多次，每次用不同的、数学上不等的值。

```cpp
int y {5};
y = y + 1;
```

变量 `y` 初始化为 5，因此表达式 `y + 1` 产生 6。这个结果被存回 `y`，效果是将 `y` 递增 1。这在普通数学中没有意义：任何数学家都会告诉你，`y` 永远不可能等于 `y + 1`（当然当 `y` 等于无穷大时除外……）。但在 C++ 等编程语言中，反复将变量递增 1 实际上极其常见。在第 5 章中，你会发现类似的表达式在循环中无处不在。

下面是一个完整的示例程序，演示距离转换：

```cpp
// Ex2_02.cpp -- 转换距离
import <iostream>;

int main()
{
    unsigned int yards {}, feet {}, inches {};

    std::cout << "请输入以码、英尺和英寸表示的距离，"
              << "三个值以空格分隔：";
    std::cin >> yards >> feet >> inches;

    const unsigned int feet_per_yard {3};
    const unsigned int inches_per_foot {12};

    unsigned int total_inches {};
    total_inches = inches + inches_per_foot *
        (yards * feet_per_yard + feet);
    std::cout << "该距离相当于 "
              << total_inches << " 英寸。\n";

    // 将英寸距离转换为码、英尺、英寸
    std::cout << "请输入以英寸表示的距离：";
    std::cin >> total_inches;
    feet = total_inches / inches_per_foot;
    inches = total_inches % inches_per_foot;
    yards = feet / feet_per_yard;
    feet = feet % feet_per_yard;
    std::cout << "该距离相当于 "
              << yards << " 码 "
              << feet << " 英尺 "
              << inches << " 英寸。" << std::endl;
}
```

典型输出：

```
请输入以码、英尺和英寸表示的距离，三个值以空格分隔：9 2 11
该距离相当于 359 英寸。
请输入以英寸表示的距离：359
该距离相当于 9 码 2 英尺 11 英寸。
```

> :::tip
> 注意使用空白来美观地排列这些赋值语句。你可以不用空格写出相同的语句，但那样读起来就不流畅了。我们通常在每个二元运算符前后各添加一个空格，以促进代码可读性。
> :::

<a id="op-复合赋值运算符"></a>

## `op=` 复合赋值运算符

在 Ex2_02.cpp 中，有一条语句你可以更简洁地写：

```cpp
feet = feet % feet_per_yard;
```

这条语句可以使用 `op=` 赋值运算符来写。`op=` 赋值运算符，也称为**复合赋值运算符(Compound Assignment Operator)**，之所以这样称呼是因为它们由一个运算符和赋值运算符 `=` 组合而成。你可以这样写：

```cpp
feet %= feet_per_yard;
```

这与前面的语句是相同的操作。

一般来说，`op=` 赋值的格式如下：

```
lhs op= rhs;
```

`lhs` 表示某种变量，是运算符结果的目的地。`rhs` 是任何表达式。这等效于：

```
lhs = lhs op (rhs);
```

括号很重要，因为你可以写出这样的语句：

```cpp
x *= y + 1; // 等效于 x = x * (y + 1)，而不是 x = x * y + 1
```

如果没有隐含的括号，`x` 中存储的值将是 `x*y+1` 的结果，这完全不同。

**表 2-3. `op=` 赋值运算符**

| 运算 | 运算符 | 运算 | 运算符 |
| --- | --- | --- | --- |
| 加法(Addition) | `+=` | 按位与(Bitwise AND) | `&=` |
| 减法(Subtraction) | `-=` | 按位或(Bitwise OR) | `\|=` |
| 乘法(Multiplication) | `*=` | 按位异或(Bitwise XOR) | `^=` |
| 除法(Division) | `/=` | 左移(Shift left) | `<<=` |
| 取模(Modulus) | `%=` | 右移(Shift right) | `>>=` |

> :::caution
> `op` 和 `=` 之间不能有空格。`x + = y` 是错误的。
> :::

<a id="sizeof-运算符"></a>

## `sizeof` 运算符

**`sizeof` 运算符**用于获取类型、变量或表达式结果所占用的字节数：

```cpp
int height {74};
std::cout << sizeof height << " bytes.\n";           // 变量
std::cout << sizeof(long long) << " bytes.\n";       // 类型
std::cout << sizeof(height * height / 2) << " bytes.\n"; // 表达式
```

你可以对任何基本类型、类类型或指针类型应用 `sizeof`。`sizeof` 产生的结果类型为 **`size_t`**——这是在标准库 `<cstddef>` 模块中定义的**无符号整数类型(Unsigned Integer Type)**。`size_t` 类型是**实现定义的(Implementation-defined)**，但如果你使用 `size_t`，你的代码将适用于任何编译器。

> :::tip
> - `sizeof` 返回 `size_t` 类型的值（在 `<cstddef>` 中定义的无符号整数类型）
> - `size_t` 是**实现定义的(Implementation-defined)**，会根据系统架构（32 位 vs 64 位）进行调整
> - `sizeof` 在**编译期(Compile Time)**求值
> - `sizeof(height * height / 2)` 其中 `height` 是 `int` 类型，结果为 `sizeof(int)`
> :::

<a id="自增与自减"></a>

## 自增与自减

你已经见过如何使用 `+=` 运算符递增变量，我们也确信你已经推断出可以使用 `-=` 递减变量。还有另外两个运算符可以完成相同的任务。它们称为**自增运算符(Increment Operator)**和**自减运算符(Decrement Operator)**，分别是 `++` 和 `--`。

这些运算符不仅仅是其他选项。你会经常见到它们，一旦深入 C++，你会发现它们非常有价值。特别是在第 5 章处理数组和循环时，你会频繁使用它们。自增和自减运算符是**一元运算符(Unary Operator)**，可以应用于整数变量。以下三条修改 `count` 的语句具有完全相同的效果：

```cpp
int count {5};
count = count + 1; // 传统方式
count += 1;        // 复合赋值
++count;           // 前缀自增
```

每条语句都将 `count` 递增 1。使用自增运算符显然最简洁。这个运算符的动作与其他你见过的运算符不同——它直接修改其操作数的值。在表达式中的作用是：递增变量的值，然后在表达式中使用递增后的值。

例如，假设 `count` 的值为 5，你执行：

```cpp
int count {5};
int total = ++count + 6; // count 变为 6，total = 12
```

自增和自减运算符在表达式中优先于任何其他二元算术运算符执行。因此，`count` 将被递增为 6，然后该值将用于评估赋值右侧的表达式。因此 `total` 将被赋值为 12。

**前缀形式(Prefix Form)**（`++count`）是先递增变量，然后在表达式中使用递增后的值。

<a id="后缀自增自减"></a>

## 后缀自增/自减

**后缀形式(Postfix Form)**（`count++`）先使用当前值，然后再递增：

```cpp
int count {5};
int total = count++ + 6; // total = 11，然后 count 变为 6
```

这等价于：

```cpp
int total = count + 6;
++count;
```

在像 `a+++b` 或 `a+++++b` 这样的表达式中，含义不太明显，编译器会怎么做也不清楚。这两个表达式实际上是相同的，但在第二种情况下，你可能意思是 `a++ + ++b`，这与前两个表达式的结果不同——它比前两个表达式的结果多 1。更清晰的写法应该是：

```cpp
total = 6 + count++;
```

或者使用括号：

```cpp
total = (count++) + 6;
```

自减运算符的规则同样适用。例如，假设 `count` 的初始值为 5：

```cpp
total = --count + 6; // total 被赋值为 10
```

然而，考虑这条语句：

```cpp
total = 6 + count--; // total 被设为 11
```

你应该小心不要在表达式中对同一变量应用这些运算符多次。假设 `count` 的值为 5，你写：

```cpp
total = ++count * 3 + count++ * 5; // 未定义行为
```

这条语句的结果是**未定义的(Undefined)**，因为该语句使用自增运算符多次修改 `count` 的值。即使根据 C++ 标准这个表达式是未定义的，这并不意味着编译器不会编译它们。它只是意味着不能保证结果的一致性。

现在考虑以下语句：

```cpp
k = k++ + 5; // 未定义行为
```

在右侧的表达式中，你递增了也出现在赋值运算符左侧的变量的值。所以你再次修改了 `k` 的值两次。在 C++17 之前，这类语句的效果也是未定义的。随着 `k` 在此之前等于 10，我们曾在老编译器上看到过 15 和 16 作为可能的结果。

非正式地说，C++17 标准添加了规则：赋值右侧的所有**副作用(Side Effect)**（包括复合赋值、自增和自减）在评估左侧操作数和执行实际赋值之前完全提交。这意味着在 C++17 中，我们的语句应该将 `k` 从 10 变为 15。

然而，表达式何时定义或未定义的精确规则仍然很微妙，即使在 C++17 及更高版本中也是如此。此外，根据经验，并非所有编译器都完全尊重 C++17 的新求值顺序规则。因此我们的建议保持不变：

> :::tip
> **在一个语句中只修改变量一次**，并且仅在确定其新值时才访问变量的先前值——也就是说，不要在同一语句中在修改变量后再次读取它。
> :::

> :::caution
> **不要在单个语句中多次修改变量。** 以下是未定义行为：
>
> ```cpp
> total = ++count * 3 + count++ * 5; // 未定义行为
> ```
>
> 同样，`k = k++ + 5;` 也是未定义的。即使 C++17 添加了赋值右侧的所有副作用在评估左侧之前完全提交的规则，精确的规则仍然很微妙。我们的建议：在一个语句中只修改变量一次。
> :::

<a id="定义浮点变量"></a>

## 定义浮点变量

当你需要使用非整数值时，就使用**浮点变量(Floating-Point Variable)**。有三种浮点数据类型：

**表 2-4. 浮点数据类型**

| 数据类型 | 描述 |
| --- | --- |
| `float` | 单精度浮点值 |
| `double` | 双精度浮点值 |
| `long double` | 扩展双精度浮点值 |

> :::note
> 你不能对浮点类型使用 `unsigned` 或 `signed` 修饰符；浮点类型始终是有符号的。
> :::

如第 1 章所述，**精度(Precision)** 指的是尾数中的有效数字位数。这些类型按精度递增的顺序排列，`float` 提供最低的尾数位数，`long double` 最高。精度只决定尾数中的位数。一个类型可以表示的数值范围由指数的可能范围决定。

C++ 标准没有规定精度和数值范围，因此每种类型你得到什么取决于你的编译器。而这又取决于你的计算机使用的是什么类型的处理器以及它使用的浮点表示法。标准确实保证 `long double` 类型提供的精度不低于 `double` 类型，`double` 类型的精度不低于 `float` 类型。

然而，如今几乎所有编译器和计算机架构都使用我们第 1 章介绍的 IEEE 标准规定的浮点数和算术运算。通常，`float` 提供七位十进制数字的精度（23 位尾数），`double` 接近 16 位（52 位尾数）。对于 `long double`，这取决于你的编译器：在大多数主要编译器上，它提供约 18 到 19 位数字的精度（64 位尾数），但在其他编译器（尤其是 Microsoft Visual C++）上，`long double` 的精度与 `double` 相同。表 2-5 显示了在 Intel 处理器上可以用浮点类型表示的典型值范围。

**表 2-5. 浮点类型范围**

| 类型 | 精度（十进制位数） | 尾数位数 | 范围（正或负） |
| --- | --- | --- | --- |
| `float` | 7 | 23 | ±1.18 × 10⁻³⁸ 到 ±3.4 × 10³⁸ |
| `double` | 15（接近 16） | 52 | ±2.22 × 10⁻³⁰⁸ 到 ±1.8 × 10³⁰⁸ |
| `long double` | 18-19 | 64 | ±3.65 × 10⁻⁴⁹³² 到 ±1.18 × 10⁴⁹³² |

表 2-5 中的精度数字是近似值。每种类型都可以精确表示零，但零和正或负范围下限之间的值无法表示，因此下限是最小的可能非零值。低于该值，就会发生**下溢(Underflow)**，值变为 0。

> :::tip
> 在某些编译器（尤其是 MSVC）中，`long double` 的精度与 `double` 相同。
> :::

示例定义：

```cpp
float pi {3.1415926f};                    // f 后缀表示 float
double inches_to_mm {25.4};               // 默认 double
long double root2 {1.4142135623730950488L}; // L 后缀表示 long double
```

`double` 类型在大多数情况下已经绰绰有余。通常只有在速度或数据大小真正关键时才使用 `float`。不过，如果你确实使用 `float`，则必须始终保持警惕，确保精度损失对你的应用来说是可以接受的。

<a id="浮点字面量"></a>

## 浮点字面量

从上一节的代码片段中可以看到，`float` 字面量附加 `f`（或 `F`），`long double` 字面量附加 `L`（或 `l`）。没有后缀的浮点字面量是 `double` 类型。浮点字面量包含小数点或指数，或两者都有；既没有小数点也没有指数的数字字面量是整数。

**指数表示法(Exponent Notation)** 使用 `e` 或 `E`：

```
5E3       （5000.0）
100.5E2   （10050.0）
2.5e-3    （0.0025）
-0.1E-3L  （-0.0001L）
.345e1F   （3.45F）
```

每个带指数的字面量后面括号中的值是等效的不带指数的字面量。当你需要表示非常小或非常大的值时，指数特别有用。

> :::note
> 大多数编译器会愉快地用缺少适当 `F` 或 `L` 后缀的字面量，甚至用整数字面量来初始化浮点变量。但如果字面值超出变量类型的可表示范围，你的编译器应该会至少发出关于窄化转换的警告。
> :::

<a id="浮点运算"></a>

## 浮点运算

浮点运算的写法与整数运算相同：

```cpp
const double pi {3.141592653589793};
double a {0.2};  // 披萨厚度（英寸）
double z {9};    // 披萨半径（英寸）
double volume {};
volume = pi * z * z * a;
```

**取模运算符 `%` 不能**用于浮点操作数，但你看到的所有其他二元算术运算符 `+`、`-`、`*` 和 `/` 都可以。你也可以对浮点变量应用前缀和后缀自增和自减运算符 `++` 和 `--`，效果与整数 essentially 相同——变量将递增或递减 1.0。

<a id="数学常量"></a>

## 数学常量

在之前的例子中，我们用自定义的常量 `pi` 计算了纽约式披萨的体积：

```cpp
const double pi {3.141592653589793}; // 圆的周长与直径之比
```

但这个数字还有其他用途。你可以用它来计算那不勒斯式披萨、加州式披萨、芝加哥式披萨、希腊披萨等的体积——尽管不是西西里式或底特律式披萨。事实上，有些人甚至把这个数字用在意大利美食之外的其他地方。数学家们称它为**阿基米德常数(Archimedes' Constant)**，通常用希腊字母 π 表示。

鉴于它在意大利美食和其他科学计算中的许多用途，如果每个开发者都必须一遍又一遍地重新发明这个轮子（或者说，一遍又一遍地重新发明这个轮的周长与其直径之比），那将是一件遗憾的事。因此，在 C++20 标准库中，他们（终于）提供了 `<numbers>` 模块，该模块定义了这个数字以及其他几个常见的数学常量。表 2-6 列出了一些最著名的示例。

**表 2-6. `<numbers>` 模块中的数值常量示例**

| 常量 | 描述 | 近似值 |
| --- | --- | --- |
| `std::numbers::e` | 自然对数的底数 | 2.71828... |
| `std::numbers::pi` | 圆周率 π | 3.14159... |
| `std::numbers::sqrt2` | 2 的平方根 | 1.41421... |
| `std::numbers::phi` | 黄金分割常数 | 1.618... |

所有这些常量的类型都是 `double`，并且对于双精度浮点数来说尽可能准确（约 17 位十进制数字）。如果你需要在 `float` 或 `long double` 精度的计算中使用这些常量——尤其是后者——你应该使用 `std::numbers::pi_v<float>` 或 `std::numbers::sqrt2_v<long double>` 形式的表达式。也就是说：在常量名后附加 `_v<T>`，将 `T` 替换为所需的浮点类型。

> :::tip
> 优先使用这些预定义的常量，而不是自己定义的常量。如果你确实需要定义一些新常量，请确保使用适当的精度。我们遇到过太多遗留代码将 π 定义为 `3.14159`，导致不必要的不准确结果！
> :::

<a id="数学函数"></a>

## 数学函数

标准库头文件 **`<cmath>`** 定义了大量你可以用于程序的三角函数和数值函数。表 2-7 展示了其中一些最有用的函数。

**表 2-7. `<cmath>` 头文件中的数值函数**

| 函数 | 描述 |
| --- | --- |
| `abs(arg)` | 计算 `arg` 的绝对值。与大多数 `<cmath>` 函数不同，如果 `arg` 是整数，`abs()` 返回整数类型。 |
| `ceil(arg)` | 计算不小于 `arg` 的最小整数（向上取整），返回浮点值。例如 `std::ceil(2.5)` 结果为 3.0，`std::ceil(-2.5)` 结果为 -2.0。 |
| `floor(arg)` | 计算不大于 `arg` 的最大整数（向下取整），返回浮点值。例如 `std::floor(2.5)` 结果为 2.0，`std::floor(-2.5)` 结果为 -3.0。 |
| `exp(arg)` | 计算 e 的 arg 次方。 |
| `log(arg)` | 计算 `arg` 的自然对数（以 e 为底）。 |
| `log10(arg)` | 计算 `arg` 的常用对数（以 10 为底）。 |
| `pow(arg1, arg2)` | 计算 arg1 的 arg2 次幂。`std::pow(2,3)` 的结果是 8.0，`std::pow(1.5f,3)` 等于 3.375f。 |
| `sqrt(arg)` | 计算 `arg` 的平方根。 |
| `round(arg)` / `lround(arg)` / `llround(arg)` | 将 `arg` 四舍五入到最接近的整数。`round()` 的结果是浮点数，即使输入是整数；`lround()` 和 `llround()` 的结果类型分别为 `long` 和 `long long`。中间情况（即 .5）向远离零的方向舍入。换句话说，`std::lround(0.5)` 得到 1L，而 `std::round(-1.5f)` 得到 -2.0f。 |

除了这些，`<cmath>` 还提供了所有基本的三角函数（`std::cos()`、`sin()` 和 `tan()`），以及它们的反函数（`std::acos()`、`asin()` 和 `std::atan()`）。**角度始终以弧度(Radians)表示**。

所有 `<cmath>` 函数都接受可以是任何浮点或整数类型的参数。除非另有说明，浮点参数的结果类型与（最大的）输入参数类型相同，整数参数的结果类型为 `double`。

示例——计算角度的余弦值：

```cpp
double angle {1.5}; // 弧度
double cosine_value {std::cos(angle)};
```

示例——将角度转换为弧度：

```cpp
const double pi_degrees {180.0};
double angle_deg {60.0};
double tangent {std::tan(std::numbers::pi * angle_deg / pi_degrees)};
```

示例——从你的脚趾到尖塔顶部的距离：

```cpp
double height {100.0}; // 尖塔高度（英尺）
double distance {50.0}; // 距底部的距离（英尺）
double angle {std::atan(distance / height)}; // 弧度
double toe_to_tip {distance / std::sin(angle)};
// 或使用勾股定理：
double toe_to_tip2 {std::sqrt(std::pow(distance, 2) + std::pow(height, 2))};
```

> :::tip
> 形如 `std::atan(a / b)` 的表达式的问题是，通过计算除法 `a/b`，你丢失了 `a` 和 `b` 的符号信息。在一般情况下，最好调用 `std::atan2(a, b)`（也由 `<cmath>` 定义）。因为 `atan2()` 知道 `a` 和 `b` 的符号，它能够正确反映在结果角度中。
> :::

> :::caution
> `<cmath>` 是一个 C 头文件，不能作为模块导入。因此你应该使用以下 `#include` 预处理指令来使其功能在你的源文件中可用：
>
> ```cpp
> #include <cmath> // 不是：import <cmath>;
> ```
>
> C 头文件可以通过名称中的 `c` 前缀来识别（例如 `<cmath>`、`<cassert>`、`<cstddef>`、`<cstdlib>`）。
> :::

下面是一个完整的浮点示例——为快乐的鱼规划鱼塘：

```cpp
// Ex2_03.cpp -- 为快乐的鱼规划鱼塘
import <iostream>;
import <numbers>; // 用于 pi 常量
#include <cmath> // 用于平方根函数

int main()
{
    // 每 6 英寸（0.5 英尺）长的鱼需要 2 平方英尺的池塘表面积
    const double fish_factor {2.0 / 0.5}; // 每单位长度的面积
    const double inches_per_foot {12.0};

    double fish_count {};
    double fish_length {};

    std::cout << "请输入你想养的鱼的数量：";
    std::cin >> fish_count;
    std::cout << "请输入鱼的平均长度（英寸）：";
    std::cin >> fish_length;
    fish_length /= inches_per_foot; // 转换为英尺

    const double pond_area {fish_count * fish_length * fish_factor};
    const double pond_diameter {2.0 * std::sqrt(pond_area / std::numbers::pi)};

    std::cout << "养 " << fish_count
              << " 条鱼所需的鱼塘直径为 "
              << pond_diameter << " 英尺。\n";
}
```

输入 20 条平均长度为 9 英寸的鱼时：

```
请输入你想养的鱼的数量：20
请输入鱼的平均长度（英寸）：9
养 20 条鱼所需的鱼塘直径为 8.74039 英尺。
```

<a id="无效的浮点结果"></a>

## 无效的浮点结果

就 C++ 标准而言，除以零的结果是未定义的。然而，大多数计算机中的浮点运算是根据 **IEEE 754 标准**（也称为 IEC 559）实现的。因此，在实践中，编译器在除以浮点数零时通常表现相当相似。

IEEE 浮点标准定义了特殊的值，其二进制尾数全为零且指数全为一，用来表示 **+无穷大(+Infinity)** 或 **-无穷大(-Infinity)**，取决于符号。当你将正的non-zero值除以零时，结果为 +无穷大，将负值除以零则结果为 -无穷大。

IEEE 标准定义的另一个特殊浮点值称为**非数(Not-a-Number, NaN)**。这表示一个在数学上未定义的结果，例如当你将零除以零或将无穷大除以无穷大时。任何一个或两个操作数都是 NaN 的任何运算结果都是 NaN。一旦运算结果为 ±无穷大，它将污染它参与的所有后续运算。

**表 2-8. 涉及 NaN 和 ±无穷大的浮点运算**

| 运算 | 结果 | 运算 | 结果 |
| --- | --- | --- | --- |
| ±值 / 0 | ±无穷大 | 0 / 0 | NaN |
| ±无穷大 ± 值 | ±无穷大 | ±无穷大 / ±无穷大 | NaN |
| ±无穷大 × 值 | ±无穷大 | 无穷大 - 无穷大 | NaN |
| ±无穷大 / 值 | ±无穷大 | 无穷大 × 0 | NaN |

（表中的"值"指任何非零值。）

**NaN(Not a Number)** 代表数学上未定义的结果。任何涉及 NaN 的运算结果都是 NaN。一旦结果为 ±无穷大，它将污染所有后续的运算。

```cpp
double a {1.5}, b {}, c {};
double result {a / b};
std::cout << a << "/" << b << " = " << result << std::endl; // 1.5/0 = inf
std::cout << result << " + " << a << " = " << result + a << std::endl; // inf + 1.5 = inf
result = b / c;
std::cout << b << "/" << c << " = " << result << std::endl; // 0/0 = -nan(ind)
```

> :::tip
> 使用 `<cmath>` 中的 `std::isinf()` 和 `std::isnan()` 来检测特殊值。另外，`NaN != NaN` 始终求值为 `true`。
> :::

<a id="陷阱"></a>

## 陷阱

使用浮点值时，你需要了解其局限性。不经意地产生可能不准确甚至错误的结果并不难。作为第 1 章的回顾，使用浮点值时常见的错误来源包括：

1. **许多十进制值无法精确转换为二进制浮点值。** 例如，十进制的 `0.1` 在二进制中是无限循环小数。因此：

   ```cpp
   0.1 + 0.2 == 0.3; // 求值为 false！
   ```

2. **灾难性抵消(Catastrophic Cancellation)：** 减去两个几乎相同的值会丢失精度。如果你取两个在第 6 位有效数字上才开始不同的 `float` 类型值之差，结果可能只有一到两位的准确度——其余的尾数都是垃圾。

   ```
   1.000001 - 1.000000 = 0.000001  // 只有 1 位有效数字，但 float 有 7 位
   ```

3. **精度损失(Loss of Significance)：** 处理相差多个数量级的值可能导致错误。一个基本的例子是用七位精度的 `float` 类型添加两个值，其中一个值比另一个大 10⁸ 倍。无论你将较小的值添加到较大的值多少次，较大的值都不会改变。

   ```
   100 + 0.45 = 100  （0.45 完全丢失）
   ```

<a id="混合表达式与类型转换"></a>

## 混合表达式与类型转换

你可以编写包含不同类型操作数的表达式。例如，你可以这样定义存储鱼数量的变量：

```cpp
unsigned int fish_count {}; // 鱼的数量
```

鱼的数量肯定是整数，所以这很有意义。每英尺的英寸数也是整数，所以你会这样定义：

```cpp
const unsigned int inches_per_foot {12};
```

尽管变量类型不同，计算仍然可以正常工作。

从技术上讲，所有二元算术操作数都需要两个操作数类型相同。但当这不是这种情况时，编译器会安排将一个操作数的值转换为与另一个操作数相同的类型。这些称为**隐式转换(Implicit Conversion)**。工作原理是：范围较窄的类型的变量被转换为另一种变量的类型。

在每个具有不同类型操作数的运算中，编译器选择范围值类型较低的操作数作为需要转换为另一种类型的那个。实际上，它按照以下顺序对类型进行排名，从高到低：

1. `long double`
2. `double`
3. `float`
4. `unsigned long long`
5. `long long`
6. `unsigned long`
7. `long`
8. `unsigned int`
9. `int`

`char`、`signed char`、`unsigned char`、`short` 和 `unsigned short` 类型的操作数始终至少被转换为 `int` 类型。

**隐式转换可能产生意想不到的结果：**

```cpp
unsigned int x {20u};
int y {30};
std::cout << x - y << std::endl; // 输出 4294967286，不是 -10！
```

你可能期望输出 -10，但事实并非如此。输出将是 4,294,967,286！这是因为 `y` 的值被转换为 `unsigned int` 以匹配 `x` 的类型，因此减法的结果是一个无符号整数值。而 -10 无法用无符号类型表示。对于无符号整数类型，低于零总是**环绕(Wrap Around)**到最大可能的整数值。也就是说，对于 32 位 `unsigned int` 类型，-1 变成 2³² - 1 或 4,294,967,295，-2 变成 2³² - 2 或 4,294,967,293，依此类推。因此 -10 确实变成 2³² - 10，即 4,294,967,286。

> :::note
> 无符号整数减法的结果环绕到非常大正数的现象有时称为**下溢(Underflow)**。当然，也存在相反的现象，称为**上溢(Overflow)**。例如，将 `unsigned char` 值 253 和 5 相加，不会得到 258——`unsigned char` 变量能保存的最大值是 255！相反，结果将是 2，即 258 mod 256。
> :::

> :::caution
> 上溢和下溢的结果**仅对无符号整数已定义**。对于有符号整数类型的变量，超出其类型可表示范围的结果是**未定义的(Undefined)**——也就是说，它取决于你的编译器和目标计算机架构。
> :::

<a id="显式类型转换"></a>

## 显式类型转换

要显式地将表达式的值转换为给定类型，你编写如下内容：

```cpp
static_cast<要转换到的类型>(表达式)
```

`static_cast` 关键字反映了转换是**静态检查(Statically Checked)**的，即在编译代码时进行检查。稍后，当你处理类时，你会遇到**动态转换(Dynamic Cast)**，其中转换是在程序执行时动态检查的。

```cpp
double value1 {10.9};
double value2 {15.9};
int whole_number {static_cast<int>(value1) + static_cast<int>(value2)}; // 25
```

`whole_number` 的初始值是 `value1` 和 `value2` 的整数部分的和，因此它们各自被显式转换为 `int` 类型。`whole_number` 因此将具有初始值 25。注意，与整数除法一样，从浮点类型转换到整数类型使用**截断(Truncation)**。也就是说，它简单地丢弃浮点数的整个小数部分。

> :::tip
> `<cmath>` 中的 `std::round()`、`lround()` 和 `llround()` 函数允许你将浮点数四舍五入到最接近的整数。在许多情况下，这比（隐式或显式）截断效果更好。
> :::

我们前面示例中的转换不影响存储在 `value1` 和 `value2` 中的值，它们将保持为 10.9 和 15.9。转换产生的值 10 和 15 只是临时存储用于计算，然后丢弃。虽然两个转换都导致信息丢失，但编译器总是假设你知道自己在做什么，当你显式指定转换时。

这里是一个完整的示例，将十进制码转换为码、英尺和英寸：

```cpp
// Ex2_04.cpp -- 使用显式类型转换
import <iostream>;

int main()
{
    const unsigned feet_per_yard {3};
    const unsigned inches_per_foot {12};
    const unsigned inches_per_yard {feet_per_yard * inches_per_foot};

    double length {}; // 以十进制码输入的长度
    unsigned int yards {};
    unsigned int feet {};
    unsigned int inches {};

    std::cout << "请输入以十进制表示的码长：";
    std::cin >> length;

    yards = static_cast<unsigned int>(length);
    feet = static_cast<unsigned int>((length - yards) * feet_per_yard);
    inches = static_cast<unsigned int>(length * inches_per_yard) % inches_per_foot;

    std::cout << length << " 码转换为 "
              << yards << " 码 "
              << feet << " 英尺 "
              << inches << " 英寸。" << std::endl;
}
```

输出：

```
请输入以十进制表示的码长：2.75
2.75 码转换为 2 码 2 英尺 3 英寸。
```

<a id="旧式转换"></a>

## 旧式转换

在 `static_cast<>` 大约于 1998 年引入 C++ 之前（也就是非常非常久以前），显式转换是这样写的：

```cpp
(要转换到的类型)(表达式)
```

表达式的结果被转换为括号之间的类型。例如，前面示例中计算英寸的语句可以这样写：

```cpp
inches = (unsigned int)(length * inches_per_yard) % inches_per_foot;
```

这种类型的转换是 C 语言的残余，因此也称为 **C 风格转换(C-Style Cast)**。C++ 中有几种不同类型的转换现在已经区分开来，但旧式转换语法涵盖了它们全部。因此，使用旧式转换的代码更容易出错。并不总是清楚你的意图是什么，你可能得不到预期的结果。此外，圆括号与周围的（复合）表达式融合得太多了——`static_cast<>()` 运算符在视觉上容易得多。

> :::tip
> **不要在 C++ 代码中使用 C 风格转换。完毕。** 它们仍然是语言的一部分以便向后兼容，但你应该始终优先使用 `static_cast<>`。
> :::

<a id="格式化流输出"></a>

## 格式化流输出

你可以使用**流操纵符(Stream Manipulator)**来更改输出流的格式化方式，通过 `<<` 插入：

```cpp
std::cout << std::setprecision(2) << pond_diameter; // 输出 8.7
```

标准 `<ios>` 和 `<iomanip>` 模块定义了更多的流操纵符：`std::hex`（十六进制输出）、`std::scientific`（科学计数法）、`std::setw()`（用于格式化表格数据）等等。

然而，在本书中我们使用 `std::format()` 代替它，因为它更强大，能生成更紧凑和可读的代码，而且通常更快。

<a id="使用-stdformat-进行字符串格式化"></a>

## 使用 `std::format()` 进行字符串格式化

C++20 的 `<format>` 模块提供了强大的字符串格式化：

```cpp
std::cout << std::format("养 {} 条鱼所需的鱼塘直径为 {:.2f} 英尺。\n",
                         fish_count, pond_diameter);
```

第一个参数是**格式字符串(Format String)**，包含**替换字段(Replacement Fields)**（`{}`）。后续参数按从左到右的顺序与替换字段匹配。

默认情况下，`std::format()` 确保**无损往返(Lossless Round-trips)**——将格式化后的字符串转换回值会产生与原始值完全相同的值。对于 `double`，这可以产生多达 16 位十进制数字。

<a id="格式说明符"></a>

## 格式说明符

在替换字段中的冒号后添加**格式说明符(Format Specifier)**：

```cpp
std::cout << std::format("养 {} 条鱼所需的鱼塘直径为 {:.2f} 英尺。\n",
                         fish_count, pond_diameter);
```

- `{:.2}` 指定总共 **2 位有效数字**：`8.7`
- `{:.2f}` 指定 **2 位小数**（定点格式）：`8.74`

格式说明符的一般形式：

```
[[fill]align][sign][#][0][width][.precision][type]
```

方括号表示可选组件。并非所有选项都适用于所有字段类型。

| 组件 | 说明 | 示例 | 结果（对于 42） |
| --- | --- | --- | --- |
| `fill` | 填充字符（默认：空格） | `*` | `***42`（配合对齐） |
| `align` | `<` 左对齐、`>` 右对齐、`^` 居中 | `{:^}` | `  42  ` |
| `sign` | `+` 显示两种符号、空格显示正号 | `{:+.2f}` | `+42.00` |
| `#` | 备用形式（添加前缀，始终显示小数点） | `{:+#x}` | `+0x2a` |
| `0` | 数值字段零填充 | `{:06}` | `000042` |
| `width` | 最小字段宽度 | `{:10}` | `        42` |
| `.precision` | 浮点精度 | `{:.3f}` | `42.000` |
| `type` | `f` 定点、`e` 科学计数、`g` 通用、`x`/`a` 十六进制、`d` 整数、`b` 二进制 | `{:x}` | `2a` |

> :::caution
> 如果你指定了不支持的格式化选项（例如整数字段的精度），`std::format()` 会抛出异常，你的程序将终止。使用 try-catch 块进行调试：
>
> ```cpp
> try {
>     std::cout << std::format("{:.2} {}", 42, 3.14); // 错误：整数字段不能用精度
> }
> catch (const std::format_error& error) {
>     std::cout << error.what(); // "precision not allowed for this argument type"
> }
> ```
> :::

> :::note
> `fill` 和 `align` 需要 `width` 才能生效。你不能在不指定对齐的情况下指定填充字符。
> :::

<a id="格式化表格数据"></a>

## 格式化表格数据

下面是一个完整的示例，演示宽度、对齐、填充和零填充：

```cpp
// Ex2_05.cpp
import <iostream>;
import <format>;

int main()
{
    // 默认对齐：数字右对齐，其他左对齐
    std::cout << std::format("{:7}|{:7}|{:7}|{:7}|{:7}\n", 1, -.2, "str", 'c', true);

    // 左/右对齐 + 自定义填充字符
    std::cout << std::format("{:*<7}|{:*<7}|{:*>7}|{:*>7}|{:*>7}\n", 1, -.2, "str", 'c', true);

    // 居中对齐 + 数字零填充
    std::cout << std::format("{:^07}|{:^07}|{:^7}|{:^7}|{:^7}\n", 1, -.2, "str", 'c', true);
}
```

输出：

```
1      | -0.2  |str    |c    |true
1******|-0.2***|****str|******c|***true
0000001|-0000.2|  str  |  c  | true
```

<a id="数字格式化"></a>

## 数字格式化

数值字段的额外选项：

- **定点格式(Fixed-Point Formatting)** 支持类型选项：`e`、`g`、`a`
- **整数字段(Integer Fields)** 不允许 `.precision`，支持类型选项：`b`（二进制）、`x`（十六进制）
- **备用形式(`#`)**：为整数添加进制前缀（`0x`、`0b`）；为浮点数强制显示小数点
- **大写类型(`E`、`G`、`A`、`B`、`X`)**：与小写版本等价，但将所有字母大写（例如 `0X`、`INF`、`NAN`、十六进制 `A-F`）
- **符号选项(Sign Option)**：`+` 在非负数前强制显示 `+` 前缀；空格字符则显示一个空格

完整的数字格式化示例：

```cpp
// Ex2_06.cpp
import <iostream>;
import <format>;
import <numbers>;

int main()
{
    const double pi = std::numbers::pi;

    std::cout << std::format("默认: {:.2}, 定点: {:.2f}, 科学: {:.2e}, "
                             "通用: {:.2g}\n", pi, pi, pi, pi);

    std::cout << std::format("默认: {}, 二进制: {:b}, 十六进制: {:x}\n", 314, 314, 314);
    std::cout << std::format("默认: {}, 十进制: {:d}, 十六进制: {:x}\n", 'c', 'c', 'c');
    std::cout << std::format("备用十六进制: {:#x}, 二进制: {:#b}, 大写: {:#X}\n",
                             314, 314, 314);
    std::cout << std::format("强制符号: {:+}, 空格符号: {: }\n", 314, 314);
    std::cout << std::format("综合: {:*<+10.4f}, {:+#09x}\n", pi, 314);
}
```

输出：

```
默认: 3.1, 定点: 3.14, 科学: 3.14e+00, 通用: 3.1
默认: 314, 二进制: 100111010, 十六进制: 13a
默认: c, 十进制: 99, 十六进制: 63
备用十六进制: 0x13a, 二进制: 0b100111010, 大写: 0X13A
强制符号: +314, 空格符号:  314
综合: +3.1416***, +0x00013a
```

**通用格式(`g` 或 `G`)** 默认等价于定点格式，但对于非常小或非常大的数字，会自动切换到科学计数法。

<a id="参数索引"></a>

## 参数索引

你可以在替换字段中冒号的左侧添加一个**索引(Argument Index)**来指定使用哪个参数（从 0 开始计数）：

```cpp
std::cout << std::format("养 {0} 条鱼所需的鱼塘直径为 {1:.2f} 英尺。\n",
                         fish_count, pond_diameter);
```

这在保持参数原始顺序的同时，交换了输出句子中值的顺序。

> :::tip
> - 索引不是必需的；你可以简单地交换参数的顺序。
> - 索引允许你对同一个值进行多次格式化：
>
>   ```cpp
>   std::cout << std::format("默认: {0}, 二进制: {0:b}, 十六进制: {0:x}\n", 314);
>   ```
>
> - 这对于国际化特别有用，翻译人员可能会重新排列句子组件。
> :::

<a id="查找类型极限"></a>

## 查找类型极限

**`<limits>` 模块**提供了 `std::numeric_limits<T>`，用于访问编译器特定的类型信息：

```cpp
std::cout << std::numeric_limits<double>::min();   // 最小正值（浮点）/ 真正最小值（整数）
std::cout << std::numeric_limits<double>::max();   // 最大值
std::cout << std::numeric_limits<float>::lowest(); // 最小值（最负的值）
```

对于**整数类型(Integer Type)**，`min()` 返回真正的最小值（有符号类型时为负数）。对于**浮点类型(Floating-Point Type)**，`min()` 返回最小的**正(Positive)**值——使用 `lowest()` 获取最负的值。

完整示例：

```cpp
// Ex2_07.cpp
import <iostream>;
import <limits>;
import <format>;

int main()
{
    std::cout
        << std::format("short 类型的范围为 {} 到 {}\n",
                       std::numeric_limits<short>::min(),
                       std::numeric_limits<short>::max())
        << std::format("unsigned int 类型的范围为 {} 到 {}\n",
                       std::numeric_limits<unsigned int>::min(),
                       std::numeric_limits<unsigned int>::max())
        << std::format("long 类型的范围为 {} 到 {}\n",
                       std::numeric_limits<long>::min(),
                       std::numeric_limits<long>::max())
        << std::format("float 的正范围为 {} 到 {}\n",
                       std::numeric_limits<float>::min(),
                       std::numeric_limits<float>::max())
        << std::format("float 的全范围为 {} 到 {}\n",
                       std::numeric_limits<float>::lowest(),
                       std::numeric_limits<float>::max())
        << std::format("double 的正范围为 {} 到 {}\n",
                       std::numeric_limits<double>::min(),
                       std::numeric_limits<double>::max())
        << std::format("long double 的正范围为 {} 到 {}\n",
                       std::numeric_limits<long double>::min(),
                       std::numeric_limits<long double>::max());
}
```

样本输出（Windows MSVC）：

```
short 类型的范围为 -32768 到 32767
unsigned int 类型的范围为 0 到 4294967295
long 类型的范围为 -9223372036854775808 到 9223372036854775807
float 的正范围为 1.17549e-38 到 3.40282e+38
float 的全范围为 -3.40282e+38 到 3.40282e+38
double 的正范围为 2.22507e-308 到 1.79769e+308
long double 的正范围为 3.3621e-4932 到 1.18973e+4932
```

> :::tip
> 使用 `std::format()` 代替链式 `<<` 运算符会产生更干净、更易读的代码。格式字符串清楚地显示了输出结构。
> :::

<a id="查找基本类型的其他属性"></a>

## 查找基本类型的其他属性

`std::numeric_limits<T>` 提供了许多其他属性：

| 类别 | 成员 | 说明 |
| --- | --- | --- |
| **值极限** | `min()` | 最小正值（浮点）/ 真正最小值（整数） |
| | `max()` | 最大值 |
| | `lowest()` | 最小值（最负的值） |
| | `epsilon()` | 机器极小值(Machine Epsilon) |
| **基本属性** | `is_signed` | 类型是否有符号？ |
| | `is_integer` | 类型是否为整数？ |
| | `is_exact` | 类型能否精确表示值？（`int`=true，`float`=false） |
| **存储** | `digits` | 保持精度所需的二进制位数（浮点的尾数位，有符号整数的值位） |
| | `digits10` | 可以无误差表示的十进制位数 |
| | `is_iec559` | 类型是否遵循 IEEE 754？ |
| **特殊浮点值** | `has_infinity` / `infinity()` | 是否支持无穷大？ |
| | `has_quiet_NaN` / `quiet_NaN()` | 是否支持静默 NaN？ |
| **指数** | `min_exponent` | 最小负指数（二进制） |
| | `max_exponent` | 最大正指数（二进制） |

获取特殊浮点值：

```cpp
float positive_infinity = std::numeric_limits<float>::infinity();
double negative_infinity = -std::numeric_limits<double>::infinity();
long double not_a_number = std::numeric_limits<long double>::quiet_NaN();
```

NaN 有两种类型：

| 特性 | 静默 NaN (qNaN) | 信号 NaN (sNaN) |
| --- | --- | --- |
| 行为 | 保持安静，程序继续运行 | 触发浮点异常 |
| 获取方式 | `quiet_NaN()` | `signaling_NaN()` |
| 常见来源 | 0/0、inf-inf、负数开平方 | 用于检测未初始化的变量 |

> :::note
> 并非所有平台都支持 sNaN。一旦 sNaN 被使用，它会自动转换为 qNaN。
> :::

<a id="字符变量"></a>

## 字符变量

`char` 类型的变量存储单个字符代码，占用 1 字节。C++ 标准不指定基本字符集使用的字符编码（通常是 ASCII，但取决于编译器）。

```cpp
char letter;           // 未初始化——垃圾值
char yes {'Y'}, no {'N'}; // 用字符字面量初始化
char ch {33};           // 整数初始化（等价于 '!'）
```

你可以用单引号中的字符字面量或整数来初始化 `char` 变量。转义序列也可以使用：

```cpp
char hex_A {'\x41'};   // 十六进制的 'A'
char oct_A {'\101'};   // 八进制的 'A'
```

`char` 变量是数值的，参与算术运算：

```cpp
char ch {'A'};
char letter {ch + 2}; // letter 现在是 'C'
++ch;                 // ch 现在是 'B'
ch += 3;              // ch 现在是 'E'
```

当使用 `cout` 或 `format()` 写入 `char` 变量时，默认以字符形式输出。要查看数值：

```cpp
std::cout << std::format("ch 是 '{0}'，代码为 {0:#x}\n", ch);
// 输出: ch 是 'E'，代码为 0x45
```

> :::tip
> 当使用 `>>` 读取到 `char` 变量时，会存储**第一个非空白字符(First Non-Whitespace Character)**。空白字符被忽略。你无法直接将数值读入 `char`——输入 `10` 会存储字符 `'1'` 的代码（ASCII 49）。
> :::

<a id="unicode-字符"></a>

## Unicode 字符

对于超过 256 个代码的字符集（需要同时支持多种语言），**Unicode** 是答案。

### `wchar_t` —— 宽字符(Wide Characters)

```cpp
wchar_t z {L'Z'};
wchar_t cc {L'\x00E7'}; // c-cedilla (ç)
```

宽字符字面量使用 `L` 前缀。然而，`wchar_t` 的大小是**实现相关的(Implementation-Dependent)**：在 Windows 上为 16 位（UTF-16），在大多数其他平台上为 32 位（UTF-32）。这使得它不适合可移植代码。

### C++20 Unicode 类型

C++20 引入了三种可移植的 Unicode 字符类型：

| 类型 | 编码 | 大小 | 字面量前缀 |
| --- | --- | --- | --- |
| `char` | 窄（通常为 ASCII/UTF-8） | 1 字节 | （无） |
| `wchar_t` | 宽（实现相关） | 平台相关 | `L` |
| `char8_t` | UTF-8 | 1 字节 | `u8` |
| `char16_t` | UTF-16 | 2 字节 | `u` |
| `char32_t` | UTF-32 | 4 字节 | `U` |

```cpp
char8_t yen {u8'\x00A5'};    // 日元符号 (¥) 的 UTF-8
char16_t delta {u'\x0394'};  // 希腊 Delta (Δ) 的 UTF-16
char32_t ya {U'\x044f'};     // 西里尔 ya (я) 的 UTF-32
```

> :::caution
> UTF-8 和 UTF-16 是**变宽编码(Variable-Width Encodings)**。并非所有字母都能用单个字符表示：
>
> ```cpp
> char8_t delta8 {u8'Δ'}; // 错误：Δ (U+0394) 在 UTF-8 中需要 2 字节
> ```
> :::

> :::note
> `char8_t` 在 C++20 中引入。遗留代码仍使用 `char` 表示 UTF-8。新代码应优先使用 `char8_t`。
> :::

标准库为 `wchar_t` 提供了 `wcin` / `wcout`，但没有为 `char8_t`、`char16_t` 或 `char32_t` 提供标准 I/O 流的直接支持。这些数据必须转换为 `char` 或 `wchar_t` 才能进行 I/O（在第 7 章进一步讨论）。

<a id="auto-关键字"></a>

## `auto` 关键字

**`auto` 关键字**告诉编译器根据初始值推导变量的类型：

```cpp
auto m {10};               // m 的类型为 int
auto n {200UL};            // n 的类型为 unsigned long
auto p {std::numbers::pi}; // p 的类型为 double
```

你也可以使用赋值或函数式表示法：

```cpp
auto m = 10;           // m 的类型为 int
auto n = 200UL;        // n 的类型为 unsigned long
auto p(std::numbers::pi); // p 的类型为 double
```

> :::caution
> 使用 `auto` 和花括号初始化时需要小心：
>
> ```cpp
> auto m {10};           // m 的类型为 int
> auto n = {10};         // n 的类型为 std::initializer_list<int>（不是 int！）
> ```
>
> 类型 `std::initializer_list<int>` 就是你用列表得到的相同类型：`auto list = {1, 2, 3};`。
> :::

在 C++11 和 C++14 中，`auto` 的花括号初始化经常意外地推导出 `std::initializer_list`。从 C++17 开始，`auto m {10};` 正确地推导出 `int`。然而，最安全的做法是：

> :::tip
> 如果你的编译器正确支持 C++17，你可以对单个值使用花括号初始化与 `auto`（不带 `=`）。否则，要么明确声明类型，要么使用赋值/函数式表示法。**永远不要将 `auto` 与花括号初始化及 `=` 结合使用。**
> :::

<a id="练习"></a>

## 练习

**编写一个程序，提示输入两个整数值，并将它们存储在整数变量 `a` 和 `b` 中。不使用第三个变量的情况下交换 `a` 和 `b` 的值。输出 `a` 和 `b` 的值。**

一种方法是使用加法和减法：

```cpp
#include <iostream>

int main()
{
    int a{}, b{};
    std::cout << "请输入两个整数：";
    std::cin >> a >> b;

    a = a + b;
    b = a - b;
    a = a - b;

    std::cout << "a = " << a << ", b = " << b << std::endl;
}
```

另一种方法是使用异或(XOR)：

```cpp
a = a ^ b;
b = a ^ b;
a = a ^ b;
```
